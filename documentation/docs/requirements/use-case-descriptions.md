# Use-Case Descriptions

### Use Case 1 - Reactions
As a user, I want to react to other's posts so I can express how much I relate to them.
1. Chatbot shows submissions
2. Everyone can view each other's posts
3. The user can react through a sidebar containing emojis by hovering over the post.

### Use Case 1 - Replies
As a user, I want to reply to other's posts so I can express how much I relate to them.
1. Chatbot shows submissions
2. Everyone can view each other's posts
3. The user can reply by hovering over the post.
4. a "reply" will appear as an option.
5. a textbox is provided to send a public or private reply. 

### Use Case 2 - Personal Use
As a user, I want to add VibeCheck to a personal messaging platform so that I can build a stronger relationship with friends
1. The user visits the repository
2. The user will be brought to a new page to select their messaging platform
3. The chatbot will request permission to join the messaging platform
4. The chabot will provide details about what it can access and store
5. Users will be given a list of commands that can be used to interact with the chatbot

### Use Case 3 - User Prompts
As a user, I want to have creative access to how I can create a prompt so that I can personalize other's responses
1. Chatbot messages the user to make a prompt
2. The user is provided a time-bound menu to make their prompt
3. The menu allows the user to choose the prompt-type, category, and must provide a statement or question

### Use Case 4 - Streak History
As a user, I want to see my streak so that I can track my consistency 
1. Every week, the chatbot messages the group chat
2. the message contains a list of all users and their streak score

### Use Case 5 - Post History
As a user, I want to see my history of past posts so that I can see who had reacted to my submissions
1. Users run a command to request a personal reply from the chatbot to see their post history
2. Chatbot sends them a list of all their posts along with the reactions and replies in the form of a private message

### Use Case 6 - Optional Selfies
As a user, I want to make selfies optional for that particular prompt so that I can still have some privacy
1. While submission UI is available, a button stating "opt-out selfie?" will be provided
2. Upon pressing the button, the user can cancel or confirm their choice
3. The submission UI will update its submission options to exclude selfie

### Use Case 7 - Music Integration
As a user, I want to add music to my post so that I can personalize and add an emotional tone 
1. While submission UI is available, the menu will have a music search bar
2. Users will select the song and a slider is provided for the user to select a specific part of the song they want
3. If no music is selected, no music will play
4. If music is selected, the most popular part of the song will play
